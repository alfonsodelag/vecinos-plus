# CLAUDE.md

## Project Context

This project is a website for:

**Vecinos Plus**

The site is being adapted from an existing website template. The objective is to convert the project into a clean, modern, friendly, practical, and conversion-oriented website for **Vecinos Plus**, without breaking the current technical architecture.

Vecinos Plus should feel like a young, reliable, approachable moving company: informal enough to feel human, organized enough to inspire trust, and practical enough to make someone request a quote without overthinking.

The website must communicate one central idea:

> Vecinos Plus helps people move, relocate, transport items, and solve logistics needs with a friendly, easygoing, responsible, and de tú a tú service.

This is not a corporate logistics website with cold language. It should feel like calling a reliable neighbor who knows the city, knows the buildings, knows the stress of moving, and shows up to help.

---

## Primary Objective

Transform the existing project into a complete website for **Vecinos Plus**, focused on:

- Residential moves
- Apartment and P.H. moves
- Office or business moves
- Local logistics
- Relocations
- Shipments / envíos
- Pickup / acarreo services
- Transport of personal items
- Transport of luggage
- Vehicle transfer, if confirmed
- Moving-related support services, if confirmed
- Quote requests through WhatsApp or contact form

Do not treat this as a generic moving company site. It should feel young, casual, helpful, real, and close — like a brand that says: “tranquilo, nosotros cargamos con lo pesado.”

### Usage Rules

- Use `#01267E` as the main brand color for headers, hero accents, trust-building sections, footer, dark buttons, navigation active states, and strong typography accents.
- Use `#F47A0E` as the main action color for CTAs, WhatsApp-style action areas, badges, highlights, icon backgrounds, hover states, and conversion moments.
- Use `#FE2400` sparingly as an energetic accent for urgency, small tags, route markers, icon details, or important visual sparks.
- Use warm off-white backgrounds to keep the site human and easygoing.
- Use white cards for service descriptions, quote forms, FAQs, and process steps.
- Avoid making the entire website dark blue. Blue should anchor trust; orange should create motion; red should spark attention.
- Do not overuse `#FE2400`. It is chili, not soup.

### Suggested Gradients

### Contrast Rule

Before finalizing buttons or text over orange/red, verify readability. White text usually works better over `#01267E`; dark blue text may work better over light orange surfaces. If using `#F47A0E` for buttons, test contrast carefully. If contrast is weak, use `#01267E` for text or darken the orange.

## Design Philosophy: Top 10 Web Design GOATs Applied

This project should synthesize the thinking of ten major web design voices into practical execution.

### 1. Ethan Marcotte — Responsive Design

Design fluidly from mobile to desktop.

Implementation rules:

- Build mobile-first layouts.
- Most quote requests will happen on mobile.
- Avoid fixed widths that break on phones.
- Use responsive grids for services and cards.
- Hero must work beautifully on small screens.
- WhatsApp CTA must remain visible and tappable.
- Images must scale without cropping important people, trucks, boxes, or context.

### 2. Jeffrey Zeldman — Web Standards and Semantic Structure

Build the site with clean, durable, standards-based markup.

Implementation rules:

- Use semantic HTML: `header`, `main`, `section`, `article`, `footer`, `nav`.
- Use accessible headings in logical order.
- Do not create div soup.
- Ensure links, buttons, forms, labels, and alt text are meaningful.
- Prioritize maintainability over cleverness.

### 3. Luke Wroblewski — Mobile-First and Conversion Clarity

The mobile experience is the real storefront.

Implementation rules:

- Place `Cotizar mudanza` and `WhatsApp` CTAs prominently on mobile.
- Reduce friction in forms.
- Keep the contact section simple.
- Use large tap targets.
- Make phone, WhatsApp, Instagram, and quote request easy to find.
- Forms should ask only what is necessary to quote.

### 4. Vitaly Friedman — Content Hierarchy and UX Depth

Make the page useful, scannable, and information-rich without becoming heavy.

Implementation rules:

- Use clear section hierarchy.
- Services must be easy to scan.
- Explain the moving process in plain Spanish.
- Add FAQs that answer real moving concerns.
- Use trust-building microcopy near CTAs.
- Avoid mystery meat navigation.
- Make quote instructions obvious.

### 5. Tobias Van Schneider — Brand Experience and Visual Atmosphere

The website must feel like a brand, not a template.

Implementation rules:

- Use the Vecinos Plus palette consistently.
- Create a memorable hero visual system.
- Use friendly spacing and confident composition.
- Build a recognizable visual rhythm: deep blue, orange, boxes, route lines, rounded cards, human copy.
- Avoid generic moving-company template clichés.

### 6. Dan Cederholm — Lightweight, Practical Interfaces

Keep components simple, fast, and durable.

Implementation rules:

- Avoid unnecessary JavaScript.
- Prefer Astro static rendering where possible.
- Keep cards and sections reusable.
- Use CSS/Tailwind utilities cleanly.
- Optimize images.
- Keep animations subtle and performant.

### 7. Nick Finck — UX Architecture and User Journeys

Design around the customer’s intent.

Primary user journeys:

1. User wants to quote a move quickly.
2. User wants to know whether Vecinos Plus handles their type of move.
3. User wants to know how the process works.
4. User wants to verify trust and professionalism.
5. User wants WhatsApp contact immediately.
6. User wants to understand what information they need before requesting a quote.

Every section should support at least one of these journeys.

### 8. Sarah Parmenter — Polished UI and Interface Details

Small details make the company feel reliable.

Implementation rules:

- Use elegant cards with consistent radius.
- Maintain consistent icon sizing.
- Keep buttons visually clear.
- Use soft hover states.
- Forms should feel quick and carefully designed.
- Use subtle visual states for focus, errors, and success.
- Make CTAs feel friendly, not aggressive.

### 9. Veerle Pieters — Grid, Illustration, and Visual Craft

Use layout discipline and tasteful visual accents.

Implementation rules:

- Use grid systems for services and benefits.
- Add subtle line-art or abstract route shapes if appropriate.
- Use box labels, tape strips, arrows, and location pins as visual motifs.
- Keep iconography consistent.
- Use decorative elements sparingly.
- Ensure illustrations do not look childish.

### 10. Mike Monteiro — Ethics and Clarity

A service website must be honest.

Implementation rules:

- Do not exaggerate capabilities.
- Do not invent coverage, fleet, staff, permits, insurance, or guarantees.
- Do not promise impossible timeframes.
- Do not use fear-based messaging.
- Make quoting requirements clear.
- Make forms transparent and respectful.
- Prioritize customer trust over aggressive conversion.
